Overview of Sorting Algorithms

Big O Notation:
A sorting algorithm is an algorithm (sequence of steps) that puts elements of a list in a certain order. The most commonly used ordering is numerical order. While there are many different types of sorting algorithms that can each be used to sort the same set of data, the efficiency and complexity of how these algorithms go about sorting the data may all be different. The efficiency of a sorting algorithm (or any algorithm in general) is measured by its "big O notation". Big O notation refers to how an algorithm's running time or space requirements grow as the input size grows. In short, big O notation characterizes functions according to their growth rates: different functions with the same growth rate may be represented using the same O notation. A list of the different big O notations (complexities) can be seen below with the most costly (least efficient) at the top of the list and the least costly (most efficient) at the bottom of the list:

  • N! → Factorial
  • 2n → Exponential
  • Nd, d > 3 → Polynomial
  • N3 → Cubic
  • N2 → Quadratic
  • N sqrt n
  • N log n
  • N → Linear
  • sqrt (n) → Root-n
  • Log n → Logarithmic
  • 1 → Constant
When I show the implementation of a given algorithm or data structure, I will also describe its bit O notation. I will refer to this list (above) so that you can see where a given complexity falls in this "efficiency list".

It should be noted that with sorting algorithms in particular, the space requirements play a factor in the complexity of the algorithm. Some sorting algorithms (slection, bubble, heapsort) work by moving elements to their final position, one at a time using only the space required for the initial array of data. For example, to sort an array of size N, first put 1 item in the proper place and then continue sorting the remaining items with the array of size N-1 at each step. These sorting algorithms do not use up an additional space compared to some other sorting algorithms (insertion, quicksort, counting, radix) that put items into a temporary position, each time closer to their final proper ordering. Each time you rescan the elements, you move the elements closer to the final position with each iteration.
